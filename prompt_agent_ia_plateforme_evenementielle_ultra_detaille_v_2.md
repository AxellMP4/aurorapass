Implémenter un système de permissions avancé basé sur un modèle RBAC strict (Role Based Access Control). Le rôle Owner correspond au propriétaire principal du compte avec accès complet à tous les paramètres, événements, analytics, gestion équipe et futures fonctionnalités business. Le rôle Event Admin peut gérer les invités, les paramètres événementiels, les QR codes, les invitations, les accès et les analytics sans pouvoir compromettre le système global du compte. Le rôle Security ou Vigile doit volontairement avoir une interface ultra simplifiée limitée au scan QR, à l’autorisation ou au refus d’accès et à la consultation des check-ins. Aucun accès administratif ne doit être disponible à ce rôle. Le rôle Guest Manager peut uniquement gérer les invités, les RSVP, les listes et les billets. Le rôle Read Only ne peut consulter les données sans les modifier. Les permissions doivent être granulaires, robustes, impossibles à contourner et vérifiées côté serveur.
